  Features 4 and 5 (Auth + User Profile) will significantly change the page 
  structure. Auth typically introduces new routes, login/signup pages, and a
  navbar with user state. The profile page is an entirely new layout. Styling
  components that will be restructured, moved, or replaced is wasted effort.

  The current components are also still minimal placeholders — unstyled <ul>,
  <li>, raw <select> elements. Styling them now means styling things you'll
  likely refactor once you see how Auth and Profile fit together.

  The better order:
  1. Finish the remaining two MVP features (Auth + Profile)
  2. Then do one styling pass across the complete, stable component tree

  At that point the Implementation Plan's recommendations (dark palette,
  responsive grid, shadcn Sheet for mobile nav) will all apply cleanly since
  you'll know the full shape of the UI.
