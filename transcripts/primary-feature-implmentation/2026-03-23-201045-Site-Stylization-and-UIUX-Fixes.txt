❯ Are there any potential overflow concerns with the newly added addkey and     
logkey?                                                                         

⏺ No practical concern. JavaScript numbers are 64-bit floats with a safe integer
   ceiling of Number.MAX_SAFE_INTEGER (9,007,199,254,740,991). At one add/remove
   per second continuously, it would take roughly 285 million years to overflow.

  If you wanted to be theoretically bulletproof, you could replace both counters
   with a boolean toggle instead:

  const [addKey, setAddKey] = useState(false)
  // increment becomes:
  setAddKey((k) => !k)

  A boolean flip can never overflow. But for a game logging app, the integer
  counters are fine — this isn't worth changing.
